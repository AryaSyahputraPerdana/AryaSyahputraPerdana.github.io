Machine Learning Pipeline

Credit Risk Analysis & Loan Status Prediction

Membangun model klasifikasi berbasis Machine Learning end-to-end untuk menganalisis dan memprediksi risiko gagal bayar pada data historis pinjaman.

Alur Kerja (Pipeline) Proyek

Proyek ini mendemonstrasikan seluruh siklus hidup pengembangan model Machine Learning, berawal dari eksplorasi raw data berskala besar (lebih dari 466.000 baris data dari periode 2007-2014) hingga membangun sistem inference untuk data baru. Berikut adalah rincian tahapan yang dieksekusi dalam Jupyter Notebook:

  • 📊 1. Exploratory Data Analysis (EDA)

    Analisis dimulai dengan memeriksa struktur dataset, tipe data, dan distribusi missing values. Visualisasi menggunakan Seaborn dan Matplotlib dilakukan untuk memahami distribusi kelas target (loan_status), menganalisis fitur numerik krusial seperti loan_amnt, annual_inc, dan dti, serta melihat matriks korelasi (heatmap) antar variabel.

  • 🧹 2. Data Cleaning & Pre-processing

    Kolom yang tidak relevan dieliminasi, serta baris dengan status pinjaman yang ambigu (seperti Current) disaring keluar. Untuk menangani data kosong (NaN), dilakukan imputation secara selektif: mengisi nilai median untuk data numerik dan modus untuk data kategorikal.

  • ⚙️ 3. Feature Encoding & Scaling

    Algoritma Machine Learning memerlukan input numerik. Kolom kategorikal (seperti term, grade, purpose) ditransformasi menggunakan LabelEncoder. Selanjutnya, fitur numerik dinormalisasi menggunakan StandardScaler untuk menyamakan rentang data. Baik encoder maupun scaler disimpan menggunakan joblib (.pkl) untuk menjaga konsistensi pada saat simulasi prediksi.

  • 🤖 4. Model Training & Evaluation

    Dataset dibagi dengan proporsi 80% data latih (Train) dan 20% data uji (Test). Tiga algoritma klasifikasi—Logistic Regression, Random Forest, dan XGBoost—dilatih dan dievaluasi kemampuannya. Evaluasi berfokus pada metrik Accuracy dan ROC AUC Score untuk memastikan model dapat membedakan kelas dengan baik, bukan hanya menebak mayoritas data.

  • 🧪 5. Testing Inference (Dummy Data)

    Sebagai bukti konsep (Proof of Concept), model diuji menggunakan beberapa skenario data dummy. Data simulasi ini melewati proses pipeline yang sama (di-transformasi oleh scaler dan encoder yang telah di-load), kemudian diteruskan ke model untuk memprediksi status akhir pinjamannya (misal: Charged Off atau Fully Paid).

Perbandingan Performa Model

Berdasarkan evaluasi terhadap data uji (test set), berikut adalah perbandingan kinerja ketiga algoritma yang diimplementasikan:

Algoritma Model Akurasi ROC AUC
Logistic Regression 93.64% 0.9476
Random Forest 93.50% 0.9229
XGBoost (Best Model) 94.61% 0.9672

💡 Kesimpulan Akhir

Algoritma XGBoost memberikan hasil prediksi terbaik dengan ROC AUC tertinggi (0.9672), membuktikan keunggulan metode gradient boosting dalam menangani data tabular kompleks. Pipeline yang dibuat juga terbukti stabil, memungkinkan penyebaran (deployment) model ke dalam bentuk API dengan alur preprocessing data yang konsisten menggunakan file .pkl.
